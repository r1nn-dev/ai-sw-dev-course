# 실습 결과 보고서 — 
> 고객 서비스 에이전트 통신 패턴 구현

##  Step 1. 통신 패턴 선택
선택한 패턴: 메모리 핸드오프 (Memory Handoff)
| 질문 | 판단 |
|---|---|
| 요구사항만 알면 되는가? (X) | 오류 코드, 재현 과정, 시도한 방법 등 맥락 전체가 필요하다 |
| 지금까지의 대화 맥락이 필요한가? (X) | 기술 지원 에이전트가 대화 전체를 보고 스스로 필요한 정보를 선별해야 한다 |
| 실시간으로 같은 메모리를 공유해야 하는가? (X) | 결과만 받으면 되므로 완전 공유 메모리까지는 불필요하다 |


## Step 2. call_agent 구현 포인트
- **메모리 격리**: 
    - 서브 에이전트는 빈 `messages`에서 시작 
    - 호출자의 이전 대화가 자동 공유되지 않는다.
- **컨텍스트 통제**: 
    - `AGENT_REGISTRY`에 등록된 에이전트만 호출 가능하다.
    - 레지스트리 내부 구조는 서브 에이전트에 노출하지 않는다.
- **결과 추출**: `result.content[-1].text` 
    - 마지막 content 블록이 최종 답변이다.
- **Tool Use**: 
    - 에이전트가 스스로 일반/기술 문의를 판단하고 `call_agent` 도구를 호출한다.
    - 하드코딩 분기가 아니다.

## Step 3. 테스트 결과
### 테스트 1 — 일반 문의
- **입력:**
    - 사용자: 영업시간이 어떻게 되나요?
- **출력:**

    ```
    [고객 서비스] 안녕하세요! 영업시간 문의 주셔서 감사합니다.

    죄송하지만 현재 저에게 제공된 영업시간 정보가 없어 정확한 안내가 어렵습니다. 보다 정확한 정보를 원하신다면 아래 방법을 이용해 주세요:

    - 이메일 문의: [현재 계정 이메일 - 개인정보로인해 지움]
    - 홈페이지 방문: 공식 웹사이트에서 확인

    추가로 궁금하신 점이 있으시면 언제든지 말씀해 주세요!
     ```
- **확인:**
    - `[고객 서비스]` 로 시작하는 답변이 반환되었다.
    - `[에이전트 판단]` 핸드오프 로그가 출력되지 않았다.


### 테스트 2 — 기술 문의

- **입력:**
    - 사용자: 앱을 설치했는데 로그인 화면에서 ERR_5043 오류가 계속 나요.

- **출력:**

     ```
    [기술 지원팀]                                                  
    고객님, ERR_5043 오류 확인했습니다. 로그인 화면에서 반복 발생하는 인증/네트워크 관련 오류입니다. 아래 순서대로 시도해 보세요.                                                                                                                                                
    ---                                                                                                                                    
    해결 절차                                                                                                                                                                                                                                                                  
    1단계. 앱 캐시 및 데이터 초기화
    - Android: 설정 → 앱 → 해당 앱 → 저장공간 → 캐시 삭제 / 데이터 삭제
    - iOS: 앱 완전 삭제 후 App Store에서 재설치

    2단계. 네트워크 환경 전환
    - Wi-Fi ↔ 모바일 데이터 전환 후 재시도
    - VPN 사용 중이라면 비활성화 후 재시도

    3단계. 앱 업데이트 확인
    - 최신 버전인지 확인 후 업데이트

    4단계. 기기 재시작
    - 완전히 껐다 켠 후 재시도

    ---
    추가 확인 사항

    더 정확한 진단을 위해 아래 정보를 알려주시면 도움이 됩니다.
    1. 기기 종류 및 OS 버전 (예: 갤럭시 S24 / Android 14, iPhone 15 / iOS 17)
    2. 오류 화면에 표시되는 전체 메시지 내용
    3. 처음 설치인지, 이전에 정상 사용한 적 있는지 여부
    4. 최근 OS 또는 앱 업데이트 진행 여부

    위 절차 시도 후에도 오류가 지속되면 다시 말씀해 주세요!

    ```

- **확인:**
    - Task Agent(기술 지원 에이전트)가 호출되었다.
    - [기술 지원팀] 으로 시작하는 답변이 반환되었다.
    - 답변 안에 ERR_5043 이 언급되었다. → 맥락이 정상 전달된 것 확인

## 완성 후 확인 
기술 에이전트가 고객 대화 맥락을 활용하는가?
(테스트 2 결과를 붙여넣고 ERR_5043 언급 여부 서술)

**결론:**
```
기술 문의 시 고객 서비스 에이전트가 스스로 판단하여 Task Agent를 호출했고,
전달받은 대화 맥락(ERR_5043)을 기술 지원 에이전트가 직접 언급하며 구체적인 해결 방법을 제시했다.
메모리 핸드오프 패턴이 정상 동작함을 확인했다.
```
